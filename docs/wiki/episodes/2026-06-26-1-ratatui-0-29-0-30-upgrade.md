# Episode: Ratatui 0.29 → 0.30 upgrade for modern widget ecosystem

## Prior State

Ratatui 0.29 with crossterm 0.28; all actively maintained Ratatui widgets (tui-tree-widget, tui-scrollview, tui-popup, tui-logger) require 0.30 post-Dec-2025 monolith split; 29er blocked from using ecosystem

## Trigger

User asks 'why don't we just upgrade to 0.30'; UI research agent confirms all maintained widget crates target 0.30, making upgrade necessary for UX improvements to proceed

## Decision

Upgrade ratatui from 0.29 to 0.30, crossterm from 0.28 to 0.29, replace tui-textarea with ratatui-textarea for compatibility

## Consequences

- Unlocks all modern Ratatui widgets: tui-tree-widget (hierarchies), tui-scrollview (virtualization), tui-popup (modals), tui-logger (live logs)
- Aligns with rest of monorepo (chirp-tui already on 0.30, shared widget code possible)
- Resolves duplicate unicode-width versions (0.1 + 0.2) in lockfile
- Requires API updates across codebase: frame.size() → frame.area(), Alignment → HorizontalAlignment, tui_textarea:: → ratatui_textarea::, Cursor → DataCursor
